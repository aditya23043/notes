Data Structures

Arrays

  • Ordered collection of data
  • Easy to find any element because of zero based indexing
  • Arrays are stored in contiguous memory which means each element of the array is next to one another in memory
  • Very good for reading elements
  • If you need to insert an element but the memory next to the last element is occupied, the computer needs to find another memory block which is of sufficient size. This is very inefficient

Time Complexity

Read O(1)
Insert O(n)
Delete O(n)

Linked Lists

  • Slow in reading but very fast in inserting and deleting elements
  • It has a pointer which has the address of the next element
  • Solves the array insertion problem since you can place the elements anywhere in the memory and simply make the previous element point to the new one
  • However, they do not have indexes and hence are slower for finding elements

Time Complexity

Read O(n)
Insert O(1)
Delete O(1)
