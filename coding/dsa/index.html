Data Type

  • Defines a certain domain of values
  • Defined operations allowed on those values

User Defined Data Types

  • Not specified in the language itself
  • Defined by the user
  • Eg: strucs, union, enums

Abstract Data Types

An abstraction of a data structure which provides only the interface to which a data structure must adhere to. The interface does not give any specific details about how something should be implemented or in which programming language.

  • User defined data types which defines operations on values using functions without specifying what is there inside the functionn and how the operations are performed (ABSTRACTION!!)
  • The person using ADTs need not know how the data type is working. Rather they are interested in how to use the data type
  • The program which uses the data structure is the called a client program
  • It has access to the ADT i.e. the interface
  • The program which implements the data strcuture is known as the implementation

Examples

Abstraction Implementation
List Dynamic Array / Linked List
Queue Linked list / Stack / Array
Map Tree map / Hash map / Hash table

Data Structures

A data structure is a way of organizing data so that it can be used effectively.

Popular uses of data structures

  • Undo and redo feature of Google Docs uses stack data structure
  • Bitmap images are stored in arrays
  • Relation information in social networking apps / sites is stored using graphs

Advantages of Data Structures

  • Efficiency
  • Reusability
  • Abstraction

Types of Data Structures

Linear DS

  • All the elements in the DS are arranged in a linear (sequential) order
  • Eg: Array

Non-Linear DS

  • No linear arrangement of the elements
  • Eg: BST

Static DS

  • Memory is allocated at compile time
  • maximum size is fixed
  • Fast Access
  • Slower insertion and deletion
  • Eg: Array

Dynamic DS

  • Memory is allocated at run time
  • maximum size is flexible
  • Slower Access
  • Faster insertion and deletion
  • Eg: Linked List

Time Complexity

Assists in comparing data strcutures in order to select the best possible DS for a particular problem / operation

Big-O Notation

Gives an upper bound of the complexity in the worst case, helping to quantify performance as the input size becomes arbitrarily large

Type Complexity
Constant O(1)
Logarithmic O(logn)
Linear O(n)
Linearithmic O(nlogn)
Quadric O(n^2)
Cubic O(n^3)
Exponential O(b^n)
Factorial O(n!)

Big-O Properties

  • O(n + c) = O(n)
  • O(c * n) = O(n) : c > 0

Big-O examples

  • Finding all subsets of a set : O(2^n)
  • Finding all permutations of a string : O(n!)
  • Soring using merge sort : O(nlogn)
  • Iterating over all cells of an n x m matrix : O(n*m)
